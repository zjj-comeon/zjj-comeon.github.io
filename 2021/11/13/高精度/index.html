优化：压位高精度

学习于此

其实我们以上的高精度，就是在一位一位的进行操作，一旦位数变多，就会降低效率。而压位高精度可以看作一种让我们多位多位的操作的方法。也可以看作我们将之从10进制转为了100甚至10000进制。

例如：

int a[] = {9,8,7,6,5,4,3,2,1};

改为：

int a[] ={9876,5432,1};

我们将实现一个完整的类，以做出更好的高精度计算，采用万进制。

我们采用vector作为容器，并有一个sign来标识正负。1表示正数，0表示负数，而数字0我们将使用sign = 1,v.size() = 1来表示。

我们需要实现：

  2. 字符串输入输出
  3. 加减乘除

除法：

除法很难算。

假设被除数为a4a3a2a1，除数为b3b2b1。那么我们只需要用 $\frac{a_4base+a_3}{b_3+b_2base^{-1} + (b_1+1)base^{-2}}$来估算一位，最后套一个加法即可。由于除数使用3位精度，能保证估计的商q’与实际的商之间的关系满足：$q-1\le q^{‘} \le q$也就是说每个位最差的时候需要试两次。但是这个时候就出现了问题，也就是$base^3$的精度需要在double之内，即：$base^3 < 2^{53}$，也就是不建议超过32768进制。

举个例子：395081 / 9876

  1. 首先试商$3950/988= 3$，于是$395081 - (9876310^1) = 98801$，这一步出现了误差。
  2. 对余数继续试商得到$9880/98 = 10$，于是$98801-(98761010^0) = 41$最终余数出现。
  3. 把试商的结果加起来并处理进位，即$310^1+1010^0 = 40$。
