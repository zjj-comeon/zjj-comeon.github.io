3、使用MySQL

use [数据库名]

在连接到数据库之后，我们需要使用USE指令（要注意MySQL的语句必须有结尾的 ‘ ; ’ ）打开一个数据库，输入

USE information_schema;将有以下输出：

这个代表我们现在打开了information_schema这个数据库。必须先使用USE指令打开数据库才可以读取其中的数据。

show databases

这一条命令可以让列出所有的数据库名。

show tables

这一条命令可以列出一个数据库内所有的表。

show columns from [表名] 和 describe

例如我们要查看目前这个数据库中keywords这个表的所有列，每个字段返回一行，行中包括字段名，数据类型，是否允许NULL，键信息，默认值以及其他信息：

Extra中有一种自动增量(auto_increment)类型，这个的意思就是为每一个行分配一个唯一的数字并且会自动递增

4、检索数据

select

使用select至少需要给出两个信息：要选择什么，从什么地方选择

select [列名] from [表名]

例如我们从products中选择一个prod_name的列：

select [列名1]，[列名2]…… from [表名]

select * from [表名]

*代表通配符，将所有的列都匹配

select distinct [列名] from [表名]

这个只返回不同的值，注意和上面的vend_id比较。

要注意的是，distinct应用于所有列而非部分列，使用不当会出现奇怪的结果：

select [列名] from [表名] limit [起始 = 0] [最大长度]

这条指令将返回从起始（默认为0）开始最长为最大长度以内的行数：

如果最大长度加上起始值大于总长度，那么将返回能返回的最大长度：

假设我们想要从第四行开始的3行：

select [表名].[列名] from [数据库名].[表名]

完全限定，有点类似于绝对路径和相对路径。

7、数据过滤

and

or

and的优先级比or更高，例如，对于一下where

where vend_id = 1002 or vend_id = 1003 and prod_price >= 10

将会先执行and语句，如果想要先执行or语句，那么请用括号将之包裹起来。

in

not

not支持对in，between和exists取反

用通配符进行过滤

利用通配符（用来匹配值的一部分的特殊字符）可以找出数据库中任意位置出现我们需要的字符的数据。（如找出所有包含anti的数据）

like

在使用通配符时要用到like操作符，like操作符指示MySQL后跟的搜索模式利用通配符而不是直接相等匹配进行比较。

%

%表示任何字符出现任意次数，如我们想要找到所有以jet开头的产品：

通配符可以在任意地方使用，并且可以存在多个。例如：

%不能匹配null

%匹配0，1或多个字符

注意尾空格，对于‘%anbil’，如果存在尾空格，那么将匹配失败。好的办法是在后面加一个%或者使用函数去掉尾空格

__

下划线匹配一个任意字符

对比%匹配：

  • 不要过度使用，如果其他操作可以达到相同的目的，那么选择其他操作符
  • 除非必要，否则不要将他们用在搜索模式的开始处，这是最慢的
  • 要注意通配符的位置

匹配字符类

一种预定义的字符集，称为字符类，方便匹配：

类 说明
[:alnum:] 任意字母和数字（[a-zA-Z0-9]）
[:alpha:] 任意字符（[a-zA-Z]）
[:blank:] 空格和制表（[\\t]）
[:cntrl:] ASCII控制字符（ASCII 0到31 和127)
[:digit:] 任意数字（[0-9]）
[:graph:] 与[:print:]相同，但是不包括空格
[:lower:] 任意小写字符（[a-z]）
[:print:] 任意可打印字符
[:punct:] 既不在[:alpha:]又不在[:cntrl:]中的任意字符
[:space:] 包括空格在内的任意空白字符（[\\f\\n\\r\\t\\v]）
[:upper:] 任意大写字母（[A-Z]）
[:xdigit:] 任意十六进制数字（[a-fA-F0-9]）

匹配多个实例

元字符 说明
* 0或多个匹配
+ 1或多个匹配（{1，}）
? 0或一个匹配（{0，1}）
{n} 指定数目的匹配
{n,} 不少于指定数目的匹配
{n,m} 匹配数目的范围（m不超过255）
