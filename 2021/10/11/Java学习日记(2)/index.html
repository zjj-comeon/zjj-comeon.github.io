类（class）和对象（object）

类是OOP（object-oriented programming）思想最重要的表现形式，他是构造对象的模板或者蓝图。类中存在字段和方法。

通俗的说，可以这样认为：一只名叫旺财的哈士奇，它属于狗类这个抽象的、共性的类别，它本身又是狗类的一个实例。它的种类（哈士奇）、名称（旺财）等等这些都是狗类的字段，对于每一个实例来说可以相同也可以不同;而狗可以叫，可以跑，可以吃饭，这些都是狗类的方法，他们对于每一个具体的狗来说，抽象出来的动作是类似的，只是不同的狗会有不同的细微的差别。

面向对象之中存在三个特征：

  1. 封装：保护内部的操作不被破坏（你永远无法改变狗的种类等字段，请不要举名字为例……）
  2. 继承：在原本的基础之上继续进行扩充（狗狗生下了小狗狗，小狗狗继承了狗妈妈的所有字段和方法，也可能会出现自己新的方法和字段）
  3. 多态：在一个指定的范围之内进行概念的转换（所有的狗，不论是什么种类，也不论我们是否知道名字等信息，我们都可以称之为狗，这称之为向上转型，直到我们查阅资料或联系狗主人之后，才能具体的说出它的字段等信息。）

类之间存在三种关系：

  1. 依赖（”use-a“），某个类的方法使用另外一个类
  2. 聚合（”has-a“），某个类的字段之中有另一个类
  3. 继承（”is-a“），即上述的继承关系

使用预定义类

要想使用类，首先需要构造对象，指定其初始状态，然后对其应用方法

对于Java中的Date来说，如果我们想要构造一个Date实例，可以这么做

new Date()

Date()是构造器（constructor，构造函数），我们构造一个实例需要使用new操作符。如果我们想要将之保存起来，则需要一个变量：

Date birthday = new Date();

更改器方法与访问器方法：

所谓更改器方法，就是说调用之后，类的字段状态会发生变化，相应的，访问器方法调用之后并不会改变对象的字段状态，而这种区别并没有类似于c++之中的const后缀，只是从方法造成的改变来说的。

  1. 构造器

    • 构造器永远与类同名，没有返回值，每次构造类的对象时都会调用将实例的字段合适的初始化。
    • 构造器永远与new运算符一起使用，不可以对一个已存在已构造完成的对象调用构造器方法。
    • 每一个类的构造器的数量都大于等于1
    • 构造器可以有任意个参数

  2. var

    当我们可以从变量的初始值推导出他们的类型时，可以直接使用var关键字声明局部变量，如：

    Employee harry = new Employee("Harry Hacker",50000,1989,10,1);等价于

    var harry = new Employee("Harry Hacker",50000,1989,10,1);

对于数值类型（int，long，double），请尽量不要使用var，因为有时难以区分0，0L和0.0之间的区别

  1. null引用

    如前面所说，一个对象要么包含一个对象的引用，要么包含一个null（表示没有任何引用对象）

    对于null引用我们不可以对其调用任何方法，否则会出现严重的错误，导致程序终止

    我们如何保证类中的类字段不被初始化为null呢？以name为例：

    • if(n==null) name = "unknown";else name = n;
    • name = Objects.requireNonNullElse(n,unknown);

    下面这种方法更加严格，如果出现了null，那么将会产生异常

    • Objects.requireNonNull(n,"The name cann't be null); name = n;
