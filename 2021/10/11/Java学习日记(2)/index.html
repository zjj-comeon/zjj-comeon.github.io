类（class）和对象（object）

类是OOP（object-oriented programming）思想最重要的表现形式，他是构造对象的模板或者蓝图。类中存在字段和方法。

通俗的说，可以这样认为：一只名叫旺财的哈士奇，它属于狗类这个抽象的、共性的类别，它本身又是狗类的一个实例。它的种类（哈士奇）、名称（旺财）等等这些都是狗类的字段，对于每一个实例来说可以相同也可以不同;而狗可以叫，可以跑，可以吃饭，这些都是狗类的方法，他们对于每一个具体的狗来说，抽象出来的动作是类似的，只是不同的狗会有不同的细微的差别。

面向对象之中存在三个特征：

  1. 封装：保护内部的操作不被破坏（你永远无法改变狗的种类等字段，请不要举名字为例……）
  2. 继承：在原本的基础之上继续进行扩充（狗狗生下了小狗狗，小狗狗继承了狗妈妈的所有字段和方法，也可能会出现自己新的方法和字段）
  3. 多态：在一个指定的范围之内进行概念的转换（所有的狗，不论是什么种类，也不论我们是否知道名字等信息，我们都可以称之为狗，这称之为向上转型，直到我们查阅资料或联系狗主人之后，才能具体的说出它的字段等信息。）

类之间存在三种关系：

  1. 依赖（”use-a“），某个类的方法使用另外一个类
  2. 聚合（”has-a“），某个类的字段之中有另一个类
  3. 继承（”is-a“），即上述的继承关系

使用预定义类

要想使用类，首先需要构造对象，指定其初始状态，然后对其应用方法

对于Java中的Date来说，如果我们想要构造一个Date实例，可以这么做

new Date()

Date()是构造器（constructor，构造函数），我们构造一个实例需要使用new操作符。如果我们想要将之保存起来，则需要一个变量：

Date birthday = new Date();

更改器方法与访问器方法：

所谓更改器方法，就是说调用之后，类的字段状态会发生变化，相应的，访问器方法调用之后并不会改变对象的字段状态，而这种区别并没有类似于c++之中的const后缀，只是从方法造成的改变来说的。

  1. 构造器

    • 构造器永远与类同名，没有返回值，每次构造类的对象时都会调用将实例的字段合适的初始化。
    • 构造器永远与new运算符一起使用，不可以对一个已存在已构造完成的对象调用构造器方法。
    • 每一个类的构造器的数量都大于等于1
    • 构造器可以有任意个参数

  2. var

    当我们可以从变量的初始值推导出他们的类型时，可以直接使用var关键字声明局部变量，如：

    Employee harry = new Employee("Harry Hacker",50000,1989,10,1);等价于

    var harry = new Employee("Harry Hacker",50000,1989,10,1);

对于数值类型（int，long，double），请尽量不要使用var，因为有时难以区分0，0L和0.0之间的区别

  1. null引用

    如前面所说，一个对象要么包含一个对象的引用，要么包含一个null（表示没有任何引用对象）

    对于null引用我们不可以对其调用任何方法，否则会出现严重的错误，导致程序终止

    我们如何保证类中的类字段不被初始化为null呢？以name为例：

    • if(n==null) name = "unknown";else name = n;
    • name = Objects.requireNonNullElse(n,unknown);

    下面这种方法更加严格，如果出现了null，那么将会产生异常

    • Objects.requireNonNull(n,"The name cann't be null); name = n;

  3. 封装的优点

    为什么我们要封装呢？因为一旦我们将字段设置为public，这就意味着任何程序都可以随意修改这些字段，我们将失去对字段的掌握，这样也就失去了oop的精髓。

    如果我们一定要获得或者设置字段的值，需要提以下三个内容：

    • 一个私有的数据字段
    • 一个公共的字段访问器方法
    • 一个公共的字段更改器方法

  4. final实例字段

    我们可以将实例字段定义为 final（可以理解为常量），这样的字段必须在构造时初始化，而且以后无法更改。

    一般final用于基本类型或者不可变的字段，但是如果将final用在可变类上，那么final 将失去我们想象中的作用，此时final将只限制该变量不能重新引用其他该类对象，而不限制变量所引用对象的改变。

  5. 静态字段与静态方法

    • 静态字段

      如果我们将一个字段定义为static，那么每个类将只有一个这样的字段，不论我们声明多少个实例，它们都将共享这一个静态字段。

           graph BT
a[class1] --- e[static field]
b[class2] --- e
c[class3] --- e
d[...] --- e

    • 静态常量

      静态常量指的是通过static final标识的变量，如：

      public static final double PI = 3.14159265358979323846;

      而静态常量的访问不需要实例，直接通过类访问即可，如：

      Math.PI;

      要注意，静态常量完全可以声明为public，因为他无法被改变，所以即使public，也不会造成影响。

    • 静态方法

      静态方法是通过static标识的方法，如上，我们不需要使用实例调用，直接使用类调用即可

      注意！静态方法不能访问实例字段，只能访问静态字段

    方法参数

    java中对于方法的参数采用传值调用，也就是说我们在方法中使用的都是我们传入参数的一个副本，因此方法将不会对传入的参数做出改变。

    但是要注意，我们存在两种参数：

    • 基本数据类型
    • 对象引用

    当我们传入基本数据类型时，传值调用将会让我们无法改变传入的数据，但是我们传入对象引用时，将可以改变对象的属性，如：

       graph TB
a[传入参数] --> c[对象]
b[方法参数] --> c
a -.copy.-> b

    以下这个例子可以证明Java是传值调用：

       graph TB
    A[a] --保持不变--> a[Alice]
    B[b] --保持不变--> b[Bob]
    A -.copy.-> c[x]
    c --交换前--> a
    B -.copy.-> d[y]
    d --交换前--> b
    c -.交换后.-> b
    d -.交换后.-> a

    对象构造

    1. 重载

      多个方法有相同的名字不同的参数，那么这就会出现重载现象。具体选择哪个将由编译器决定。如果无法决定选择哪个或者均不符合条件，那么将报错。（这个寻找的过程称之为重载解析）

      java允许重载任何方法

      一个包含方法名和参数类型的签名可以完整的描述一个方法（注意不包含返回类型）

    2. 默认字段初始化

      如果构造器不显式地为字段设置初始值，那么就会被自动的赋值为默认值：数值为0，布尔值为false，对象引用为null

    3. 无参数的构造器

      无参数的构造器用于在用户没有提供任何信息是，将字段合理的初始化。要注意的是，如果类中不存在任何一个构造器的话，那么编译器会为我们提供一个无参数构造器；如果存在至少一个有参数构造器，那么我们必须也提供一个无参数构造器，否则不合法。

      如果我们想让类中的字段进行默认初始化的话，只需要在花括号中不编写任何代码即可。

    4. 参数名

      我们一般使用单个字母或者在每个参数前面加一个a

    5. 调用另一个构造器

      如果构造器的第一个语句形如this(...)，这个构造器将调用同一个类的另一个构造器，括号中传入另一个构造器所需要的参数。

    6. 初始化块

      在一个类中可以包含任意多个代码块，只要构造这个类的对象，这些块就会被执行。

      这样无论我们使用哪个构造器，id字段都会被初始化

      注意，如果将初始化块标记为static，那么这个代码块在类第一次加载时会进行且只进行一次。

    总的来说，初始化的处理步骤如下：

    1. 如果构造器的第一行调用了其他构造器，则基于参数执行第二个构造器
    2. 否则
      1. 所有数据字段初始化为默认值
      2. 按照类声明中出现的顺序，执行所有字段初始化方法和代码块
    3. 执行构造器主体代码

包

Java存在一种将类组织在一个集合（包）中的模式。这可以方便组织自己的代码。

  1. 包名

    使用倒写的域名+工程名+类名，如com.horstmann.corejava.Employee

    注意，从编译器的角度来看，嵌套的类之间不存在任何关系

    在文件顶部加上package packageName即可将文件加入包中

  2. 类的导入

    我们想要使用其他类时，可以使用类完整的名字，也可以通过import机制简化。我们可以通过import导入一个特定的类或者整个包。import位于文件顶部，package后。

    import java.time.*将导入java.time中所有的类，如果将*换为某个类名，则将导入一个具体的类

    只能使用*符号导入一个包，不能import java.*或者java.*.*

    如果我们导入的包中有重名的类，我们可以单独导入这个类来解决这个问题。

  3. 静态导入

    如果我们在import后面机上static，那么将成为静态导入，我们将可以直接使用其中的静态字段和方法，而无需加类名前缀

  4. 包访问

    如果标记为public，则任何类均可访问，如果标记为private，则只有自己可以访问，如果未标记，那么整个包均可访问。
