时间复杂度

时间复杂度主要用于粗略的刻画算法的运行时间，我们在计算过程中会忽略硬件的问题，只考虑代码层面所用的时间。

首先时间复杂度来自于对于代码执行次数的粗略估计，一般情况下一行代码认为是一条指令，执行一次，消耗同样的时间（除函数的调用），例如：

我们这样统计其中指令条数：

红色框中的代码执行了一次，蓝色框中的代码从i = 0一直执行到i = n，一共是n + 1次，紫色框框一共是执行了2 * n次，因此总共执行了2 * n + n + 1 + 1 = 3n + 2，程序大致和输入n（问题的规模）成线性关系。假设用T(n)来表示程序运行时间，每条指令消耗一单位时间，那么：
$$
T(n) = (3 * n + 2) * 1;
$$
但是这样的表达式过于具体，当表达式更加复杂时，我们将难以预测算法的大致运行时间，或者会因此浪费很多不必要的时间，因此选择了一种更加方便的表达形式。

我们一般只关心随着问题规模n趋于无穷大时函数中对结果影响最大的项，也就是最高次项，例如以上的公式我们将使用：
$$
T(n) = n
$$
来近似估计算法性能。

以下是算法导论中关于算法时间界限的数学描述和具体符号：
$$
\Theta(g(n)) = {f(n):存在正常量c_1、c_2和n_0，使得对所有n\geq n_0，有0 \leq c_1g(n) \leq f(n) \leq c_2g(n) }
$$
其中：
$$
O(g(n)) = c_2g(n), \Omega(g(n)) = c_2g(n).其中O代表上界，\Omega代表下界。
$$
我们这里填入的g(n)就是上面简化后的T(n)，以后将直接用O(g(n))来代表时间复杂度。

一般来说，正常的步长为一的循环，时间复杂度为O(n)，每嵌套一次，次数加一，其他请自行分析，给出复杂度的大小排序：
$$
O(1) < O(log \ n) < O(n) < O(nlog \ n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)
$$
其中log是以2为底的对数。（不论底是多少都能转为一个常数乘以一个以2为底的对数）
