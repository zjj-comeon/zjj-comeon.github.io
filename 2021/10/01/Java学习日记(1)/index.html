注释

Java中存在三种注释形式，分别是：

  1. 单行注释 // comments

    • 只注释单行

    • 注释中出现\u时要小心，\u代表之后会出现四个十六进制符号表示转义，可能会有意想不到的错误。

      // look inside c:\users会产生错误

      // \u000A is a newline 由于先进行转义序列的替换，这个\u000A会替换为换行符。

  2. 多行注释 /* comments */

    • 不可以嵌套使用该类注释，也就是说不可以简单地把代码用 /* 和 */ 括起来作为注释，因为这段代码本身也可能包含一个 */ 界定符

  3. 文档注释 + JavaDoc

    • 文档注释负责描述类，接口，方法，构造器，成员属性。可以被JDK提供的工具JavaDoc解析，自动生成一套网页形式的文档注释。
    • 标签：
      • 块标签：只能放在主要描述部分后面的标签部分。 块标签的格式为：@tag。
      • 内联标记：可以放在主要描述中的任何位置或块标签的注释中。 内联标记用花括号标记：{@ tag}。

    JavaDoc官方说明

文档注释必须写在类、接口、方法、构造器、成员字段前面，写在其他位置无效。

数据类型

Java共有8种数据类型，其中4种整形，2两种浮点型，1种字符类型，一种用于表示真值的boolean类型。

  1. 整型
    • long，8字节，-9223372036854775808~9223372036854775807
    • int，4字节， -2147483648~2147483647
    • short，2字节，-32768~32767
    • byte，1字节，-128~127

长整型：后缀L或l（如400000000L）。

十六进制：前缀0X或0x（如0xCAFE）。

八进制：前缀0，但是不推荐使用（容易混淆）。

二进制：前缀0B或0b。

另外可以在数字字面量上加下划线，如用1_000_000，表示100万。编译时会去掉这些下划线。

  1. 浮点类型
    • double，8字节，大约 $\pm$1.79769313486231570E+308（有效数位为15位）
    • float，4字节，大约 $\pm$3.40282347E+38F（有效位数为6~7位）

  • 所有的浮点数值都遵循IEEE 754规范，存在以下三个特殊的浮点数值（同样存在float类型的数值）

    • 正无穷大(Double.POSITIVE_INFINITY)
    • 负无穷大(Double.NEGATIVE_INFINITY)
    • NaN(Double.NaN)

      在我们检测的时候，不可以使用

      if (x == Double.NaN) //is never true

      所有非数值都被认为不相等，我们应该使用以下方法检测：

      if (Doubel.isNaN(X)) //Check whether x is "not a number"

      float数值后面有一个F或f，如果没有的话，那么存储为double类型

      也可以用十六进制表示浮点数，例如0.125 = 2^-3^ 可以表示为0x1.0p-3。要注意的是尾数采用十六进制，指数采用十进制，指数基数是2。

  1. 字符类型

    1. char

      • char类型的字面量需要使用单引号括起来，例如 ’A‘ 是编码值为65的字符常量，与 “A” 不同，“A”是包含一个A的字符串。char类型的值也可以表示为十六进制值。
      • Unicode转义序列会在解析代码之前得到处理，粗浅的可以理解为c语言中的#define。

    2. Unicode和char类型

      • 代码点（code point）：与一个编码表中的某个字符对应的代码值。在Unicode标准之中，代码点使用十六进制书写，并加上前缀U+，如字母A的代码点就是U+0041。
      • 代码单元：在第一代码级别中，每个字符用16位表示，而辅助字符采用一对连续的代码单元进行编码。

Unicode的代码点可以分成17个代码级别(code plane).
第一个代码级别成为基本的多语言级别,代码点从U+0000到U+FFFF,其中包括了经典的Unicode代码.
其余的16个附加级别,代码点从U+10000到U+10FFFF,其中包括了一些辅助字符(增补字符).
增补字符是用一个长度为2的char数组表示的，分别表示高代理项和低代理项。

通俗的理解，代码点就是Unicode中的各个字符，若此字符长为16位，那么有一个代码单元，若一个字符的长度为32位，则该字符有两个代码单元。

  1. boolean类型

    boolean有两个值：true和false。

java中对于布尔值的使用更加严格，不允许写出if (x = 0)这样的代码

变量与常量

  1. 声明变量

    首先指定变量类型，然后是变量类型。Java中变量名必须是以字母开头并由字母或数字组成的序列。字母包括：‘A’ ~ ’Z’ , ’a’ ~ ‘z’ , ‘_’ , ’$’或在某种语言中表示字母的任何Unicode字符，数字包括：‘0’ ~ ‘9’ 和某种语言中表示数字的任何Unicode字符。变量名大小写敏感，长度基本没有限制。

单独下划线不可以作为变量名

  1. 变量初始化

    • int x = 0;
    • int x; x = 5;

  2. 常量

    • 使用final指示该变量为常量，（将final放于类型说明符之前，如final int x = 10;）表明该变量只能赋值一次，一旦赋值就不可更改。

    • 在Java中，经常希望某个常量可以在一个类的多个方法中使用，通常将这些常量称为类常量。可以使用static final关键字设置一个类常量，前面还需要加上修饰符。相比于不带static的常量，static常量不论创建多少类的实例，都只存在一个，可以减少内存占用。

  3. 枚举类型

    有时变量的值在一个固定有限的范围内（如衣服的尺寸），将他们单纯的表示为char或者int容易出错，因此我们将其声明为枚举类型，使用一组带标识符的常数。

    • 声明枚举

现在可以这样使用枚举类型：

Size s = Size.MEDIUM;
