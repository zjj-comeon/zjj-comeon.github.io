  4. 多态

    “is - a”规则告诉我们：子类的每个对象也是超类的对象，也可以表述程序中出现超类对象的任何地方都可以使用子类对象替换。

    在Java中，对象变量是多态的，一个Employee类型的变量可以引用Employee对象以及所有由它派生而来的子类对象。

    首先声明一个Manager数组

    Manager[] managers = new Manager[10];

    然后将之转换成Employee数组完全可行：

    Employee[] staff = managers;

    接下来就是诡异的一幕了

    staff[0] = new Employee(...);

    这让一个Manager变量接受了一个Employee引用！这会导致一个ArrayStoreException。

  5. 理解方法调用

    假设我们要调用x.f(args)

    • 编译器查看对象的声明类型和方法名。如果存在多个，那么将之列举出来，至此编译器确定所有可能调用的对象。

    • 编译器确定方法调用中提供的参数类型，优先选择完全匹配的方法，这被称为重载解析。如果我们调用x.f(“hello”)，将挑选 f(String) 而非 f(int) 。由于允许类型转换，情况会很复杂，最后如果编译器找不到任何一个可以匹配的办法，或者存在多个可选的办法时，将会报错。

      1. 方法的签名包括名字与参数列表，返回类型不属于签名的一部分。
      2. 如果子类定义了一个与超类方法签名相同的方法，那么将产生覆盖

      基于以上两点，我们可以在覆盖超类方法时，将返回类型改为原返回类型的子类型，例如

      public Employee getBuddy();

      改为：

      public Manager getBuddy();

      这也成功覆盖了超类的方法。我们称之为这两个getBuddy方法有可协变的返回类型。

      要注意的是，只有在返回的是本类的超类的时候，才可以将之更改为子类对象，不可以随意更改为其他类型。

    • 如果是private、static、final标识的方法，编译器将可以准确的直到应该调用哪个方法，这叫做静态绑定，如果调用的方法依赖于隐式参数的实际类型，那么将使用动态绑定。

    • 如果采取了动态绑定，虚拟机将首先寻找x中是否存在该方法，如果不存在，那么向其超类中寻找。

      虚拟机为了减少这种寻找的时间开销，提前为每个类准备好了一个方法表。

  6. final类和方法：阻止继承

    当一个类被标记为final时，这个类将不允许被派生，也就是这个类将不存在子类，格式如下：

    public final class noSubClass

    类中的某个方法也能声明为final，这将导致该方法不可被子类覆盖（final类中的所有方法默认都是final，但是字段并非默认final）

  7. 强制类型转化

    Manager boss = (Manager) staff[0];

    这其中 staff[0] 是一个被Employee变量引用的Manager对象，我们希望将之还原为Manager，因此进行强制类型转换，这可以认为是暂时忽略对象的实际类型之后使用对象的全部功能。

    如果我们强制的在继承链上进行从上往下的转换，那么将报错

    Manager boss = (Manager) staff[1];

    这行代码强行将一个Employee转换为Manager，产生ClassCastException，如果不捕获这个异常。程序将终止。因此我们再转换前需要先查看是否能成功，只需要使用instanceof操作符即可

    我们可以在instanceof之前放入null，只是返回false

    一般很少需要强制类型转换，如果需要的话，请检查设计是否合理。

  8. 抽象类

    在类的继承层次中，超类一般位于靠上的位置，而子类更加靠下，且位于上层的类更加具有一般性，也更加抽象。我们将许多类的共性提取出来放在一个更高层次的类，这个类被称之为抽象类，由abstract标识。例如我们现在将Employee类和Manager类抽象为Person类，并且为他们增加一个getDescription方法，这个方法返回对一个人的简短描述。对于Person类，我们无法确定应该返回什么东西，因此我们将这个方法声明为abstract，这样就不需要实现了，同时，包含一个或多个抽象方法的类也需要被声明为抽象的，即：

    不含抽象方法的类也可以声明为抽象类

    抽象类也可以包含字段和具体方法，但是请尽量将通用的方法放在超类中。

    当子类继承抽象类时，可以选择定义全部方法或者保留部分或全部的抽象方法。前者的子类不再是抽象类，后者仍然是一个抽象类。

    但是，抽象类不能实例化。也就是说我们不能引用一个抽象类，可以定义一个抽象类的变量，但是只能引用它的非抽象子类。

Object

Object是所有类的始祖，我们不需要显示的指明派生于Object，因为这是默认的行为

  1. Object类型变量

    可以使用Obejct类型的变量引用任何类型的对象

    Object obj = new Employee("Harry Hacker",35000);

    当我们需要具体调用类中的一些方法时，请将之转化为对应的类型。

    Employee e = (Employee) obj;

    在Java中只有数值、字符和布尔类型的值不是对象，对于数组类型来说，不管是对象数组还是基本类型的数组都扩展了Object类。

  2. equals方法

    equals用于检测两个对象是否相等。假设我们检测两个Employee对象

    在子类中定义equals方法时，首先调用超类的equals，如果检测失败，对象就不可能相等。如果超类中的字段都相等，再去比较子类中的实例字段

  3. 相等测试与及继承

    Java语言规范要求equals方法具有以下特性：

    • 自反性：对于任何非空引用x，x.equals(x)应该返回true
    • 对称性：对于任何引用x和y，当且仅当y.equals(x)返回true时，x.equals(y)返回true
    • 传递性：对于任何引用x、y和z，如果x.equals(y)返回true，y.equals(z)返回true，x.equals(z)也应该返回true
    • 一致性：如果x和y引用的对象没有发生变化，反复调用x.equals(y)应该返回同样的结果
    • 对于任意非空引用x，x.equals(null)应该返回false。

    对于instance：

    当隐式参数和显示参数不属于同一个类时，有些程序员喜欢写

    • 由超类决定相等性概念，那么就可以使用instanceof检测

泛型数组列表

解决了Java数组在确定大小之后难以改变的问题，它叫做ArrayList，可以自动调节数组容量。

泛型类指的是我们需要给ArrayList指定我们要存储的元素类型，类型放在<>之中并追加在ArrayList的后面。

  1. 声明

    声明并构造一个数组列表

    ArrayList<Employee> staff = new ArrayList<>()'

    或者使用var：

    var staff = new ArrayList<Employee>();

    对于第一种声明方式，我们称之为菱形语法（<>），可以结合new使用，无需指定<>中的值，编译器会自动推断

    使用add可以向其中加入元素

    staff.add(new Employee("Harry Hacker",...));

    当数组满了之后会自动创建更大的数组，不过会产生一定的时间开销。如果我们预先直到可能存储的数量，可以在填充数组之前使用ensureCapacity方法

    staff.ensureCpacity(100)

    这样前100次add的调用将不会带来很大的分配空间的时间开销

    也可以在声明时的括号之中填入我们想要的初始容量

    ArrayList<Employee> staff = new ArrayList<>(100)

    等价于使用ensureCapacity

    但是要注意这不等价于

    new Employee[100];

    分配100个元素的数组，那么就存在100的空位置可以使用，而100个元素的数组列表。则只是保证前100次add不会产生巨大开销，但是在完成构造化之后可能不包含空位置。

    size方法返回保存的元素个数

    如果我们可以确定以后不会再向数组列表之中添加元素，可以使用trimToSize削减数组列表的大小（但是仍可以添加，只是产生开销）

对象包装器与自动装箱

我们需要将基本类型转换为对象时，可以使用与之对应的类：

int long double float short byte boolean char
Integer Long Double Float Short Byte Boolean Character

包装器类是不可变的，一旦构造完成，其中的值不在允许更改，也不允许被继承（定义为final）

例如，ArrayList不允许放入基本类型，我们必须用Integer包装。

var list = new ArrayList<Integer>();//不可以填入int

自动装箱的意思是：

list.add(3)

将自动变成：

list.add(Integer.valueOf(3));

自动拆箱的意思是：

int n = list.get(i);

将自动变为：

int n = list.get(i).intValue();

自动拆箱也适用于表达式，(甚至Integer可以自动拆箱升级为double之后在装箱)

Integer n = 3; n++;

对象包装器和基本类型有一点不一样，当比较两个我们看起来一样值的对象包装器的时候，其实是在比较他们的内存地址而非值，因此比较很可能失败.

有一些基本方法被放在包装器中，例如将一个数字字符串转换成数值的方法：

int x = Integer.parseInt(s);

parseInt是一个静态方法，与Integer无关。

枚举类

以下是一个枚举类的定义

public enum Size{SMALL,MEDIUM,LARGE,EXTRA_LARGE};

事实上这里声明了一个刚好有四个实例的类，不能对其构造新的对象。因此比较枚举类只需要==即可

枚举类也可以有构造韩式，但是必须声明为private或者protected

构造函数将由编译器自动调用，我们只需要调用getAbbreviation即可得到该字段

Enum超类有一个toString方法，可以返回枚举名的字符串表示。

Size.SMALL.toString();//return "SMALL"

Enum也存在一个静态方法values，可以返回包含全部枚举值的数组。

ordinal方法返回enum声明中枚举常量的位置，从0开始计数

继承的设计技巧

  1. 将公共操作和字段放在超类中
  2. 不要使用受保护的字段
    • 子类集合是无限的，任何程序员都能派生一个子类，破坏了封装性
    • 同一个包的类都能访问，破坏了封装性
  3. 使用继承实现“is-a”关系，没有明显的is-a关系时，请不要滥用！
  4. 除非所有集成的方法都有意义，否则不要继承
  5. 在覆盖方法时，不要改变预期行为
  6. 使用多态而非类型信息
