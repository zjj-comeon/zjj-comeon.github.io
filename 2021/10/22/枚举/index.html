枚举

枚举可能是计算机界翻译最准确的名词了???

枚是指的一个一个拿起，举是很多手一起（与）托起来，听完这个解释之后顿时感觉，计算机的学生也是需要很多人文气息的

枚举要做到一一例举，不重不漏。

我们如何优化枚举？答案是减少枚举的次数：

  • 使用一个符合题意的，能将范围变得最小的条件，将枚举范围缩小。例如要筛选一个男女比1：1的数据集中1月份过生日的男同学。我们首先应该使用一月这个条件来筛选，这可以瞬间将范围缩小到1/12，然后再搜索男性即可。
  • 我们需要选择好枚举的方向。例如我自认为我在班级中排名靠前，那么我就会在成绩单中从前往后寻找，而非从后往前找。
  • 选择合适的数据维护方向。我们要想清楚如何表示给定的数据对于我们来说更加方便，例如机器翻译那道题，我们之所以定义vis数组，就是因为这样维护之后可以让算法更加简单。

枚举的大致流程如下：

graph LR
a[开始] --> b{枚举值在给定的范围中吗?}
b --No--> f[结束]
b --Yes--> c{枚举值符合给定的条件吗?}
c --No--> d[取下一个枚举值]
c --Yes--> e{输出该枚举值}
e --> d
d --> b

来看这样一道题：

题目描述：

为了准备一个独特的颁奖典礼，组织者在会场的一片矩形区域（可看做是平面直角坐标系的第一象限）铺上一些矩形地毯。一共有n张地毯，编号从1到n。现在将这些地毯按照编号从小到大的顺序平行于坐标轴先后铺设，后铺的地毯覆盖在前面已经铺好的地毯之上。地毯铺设完成后，组织者想知道覆盖地面某个点的最上面的那张地毯的编号。注意：在矩形地毯边界和四个顶点上的点也算被地毯覆盖。

输入描述：

第一行，一个整数n，表示总共有n张地毯。
接下来的n行中，第i+1行表示编号i的地毯的信息，包含四个正整数a，b，g，k，每两个整数之间用一个空格隔开，分别表示铺设地毯的左下角的坐标（a，b）以及地毯在x轴和y轴方向的长度。
第n+2行包含两个正整数x和y，表示所求的地面的点的坐标（x，y）。

输出描述：

输出共1行，一个整数，表示所求的地毯的编号；若此处没有被地毯覆盖则输出-1。

样例：

输入：

3
1 0 2 3
0 2 3 3
2 1 3 3
2 2

输出：

3

一个很正常的思路就是，我们依次检测每一次是否将(x,y)这个点覆盖，如果覆盖了那么更新这个标志，如下：

我们对其优化，可以这样想：这个编号永远是最后一个放上去的地毯，那么我们为什么不从后往前遍历直到我们枚举出第一个覆盖的地毯呢？可能速度并没有快很多，但是相比于上面的枚举已经进行了优化。（因为访问数组的原因可能并没有快，这里只考虑枚举算法的优化）

在看这样一道题：

题目描述：

牛牛习惯用8位数字表示一个日期，其中，前4位代表年份，接下来2位代表月份，最后2位代表日期。

牛牛认为，一个日期是回文的，当且仅当表示这个日期的8位数字是回文的。现 在，牛牛想知道：在他指定的两个日期之间包含这两个日期本身），有多少个真实存在的日期是回文的。

一个年份是闰年当且仅当它满足下列两种情况其中的一种：

1.这个年份是4的整数倍，但不是100的整数倍；

2.这个年份是400的整数倍。

输入描述：

输入包括两行，每行包括一个8位数字。
第一行表示牛牛指定的起始日期。
第二行表示牛牛指定的终止日期。
保证datei和都是真实存在的日期，且年份部分一定为4位数字，且首位数字不为0。
保证date1 —定不晚于date2。

输出描述：

输出一行，包含一个整数，表示在date1和date2之间，有多少个日期是回文的。

样例：

输入：

20110101
20111231

输出：

1

这个题很多人第一下想到的很可能是从某年的第一天开始一直遍历到这一年的最后一天，但是其实稍微思考一下就能发现，每一年一定只有一天可能会是回文日期，于是我们直接将问题分解为从年份或者月份+日期构造回文串然后比较大小，而从年份入手的话，将会面临巨多的条件判断，因此我们从月份+日期构造。同时我们发现并不需要检测闰年，因为9220是闰年。

这个题就是对枚举优化的应用，直接降低了太多的复杂度
