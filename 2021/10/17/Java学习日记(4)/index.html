接口

  1. 概念

    接口并不是一个类，而是对希望符合这个接口的类的一组需求。

    例如Comparable接口：

    可以理解为接口是一个搭建好的类框架，我们要做的是去实现它。

    加入我们想要使用Arrays类的sort方法对Employee对象数组排序，Employee类必须实现Comparable接口，即：

    1. 将类声明为实现给定的接口

      class Employee implements Comparabla

    2. 对接口中的所有方法提供定义

    接口中所有方法自动是public，不必提供public关键字，但是在实现的时候一定要指定public

    那么，为什么我们不能在一个类的内部直接定义comparable方法呢？

    因为编译器必须能够保证这个类存在compareTo方法，如果规定为类内直接定义，难以保证每位程序员都能记得这件事情。但是如果规定必须实现comparable接口的话，那么就一定可以排序

  2. 属性

    我们不能实例化一个接口

    x = new Comparabla(. . .); //Error

    但是可以声明一个接口变量，而且可以引用实现了该接口的类对象

    Comparable x = new Employee(. . .);

    也可以使用instanceof来检测某个对象是否实现了特定的接口

    if(anObject instanceof Comparable){. . .}

    同样可以继承接口，将通用性接口继承为专用性接口

    接口中不允许存在实例字段，但是可以存在常量，所有常量默认为public static final

    超类只能有一个，但是可以实现多个接口

    class Employee implements Cloneable,Comparabla

  3. 接口与抽象类

    之所以Java要单独弄一个接口出来，而不是直接继承抽象类，是因为Java只能一次继承一个类，有了接口之后将更加方便。

    而Java之所以只能继承一个类，是因为多重继承要么很复杂，要么效率很低，所以选择用接口来实现这个做法

  4. 默认、静态和private方法

    静态方法只能由接口名称调用，private方法用于其他方法的辅助方法

    默认方法用于给接口一个默认实现

    如果先在一个接口中将一个方法定义为默认方法，然后又在超类或者另一个接口中定义同样的方法，我们有以下规则来解决这个问题：

    1. 超类优先：超类存在具体方法时，将覆盖其他方法

    2. 接口冲突：如果两个类都有一个同名且参数相同的方法，必须覆盖这个方法来解决

      类将继承两个接口的getName方法，只需要我们在Student类中提供一个getName方法即可，可以选择两个冲突方法中的一个

      即使有一个接口没有默认实现，也会报二义性错误，只要至少一个接口存在默认实现，那么就要报错

  5. 接口与回调

    回调是一种常见的设计模式，在这种模式中，可以指定某个特定事件发生时应该采取的动作。例如，按下鼠标或选择某个菜单项时，你可能希望完成某个特定的动作，这就是回调。

    Timer类可以做到经过一定时间间隔作某个动作。我们向其中传入一个实现了ActionListener接口的类，这个类将做出打印当前时间并响一声的动作，开始main函数之后，除非我们点击结束，否则程序将每隔一秒打印一次时间并响一次。

       graph LR
a[original] --> c[Employee]
b[copy] --copy--> c
d[original] --> e[Employee]
f[clone] --clone--> g[Employee]

    clone方法是Object的一个protected方法，我们的代码不能直接调用它，只有Employee可以克隆Employee对象。因为Object对这个对象一无所知，只能逐个字段拷贝，但是一旦这对象包含对一个字对象的引用，那么拷贝之后就会得到相同对象的一个引用，这样就失去了克隆的意义

    • 浅拷贝：直接拷贝克隆，这在包含不可变的子引用时，或者在对象的生命周期中子对象一直包含不变的常量，没有更改器方法会改变他，也没有方法会生成他的引用同样适用。
    • 深拷贝：对于每一个子引用也进行克隆。

    所以我们有以下选择：

    1. 默认的clone是否满足需求
    2. 是否可以在可变的子对象上调用clone来修补默认的clone方法
    3. 是否不该使用clone

    3是默认选项，1我们将实现Cloneable接口，2重定义clone方法，指定public访问修饰符

    对象对于克隆十分诡异，Cloneable内不含clone方法，但是一旦一个对象使用了clone方法却没有实现这个接口，那么将会出现异常CloneNotSupportedException

    Cloneable是一个标记接口，不含任何方法，唯一的作用是允许使用instanceof运算符。

  1. 函数式接口

    对于只有一个抽象方法的接口，需要这种接口对象的时候，就可以提供一个lambda表达式，这种接口称为函数式接口。

    对于Arrays.sort方法，第二个参数我们需要传入一个Comparator实例，Comparator就是只有一个方法的接口，所以可以提供一个lambda表达式

    Arrays.sort(words,(first,second) -> first.length() - second.length());

    也就是说lambda可以转换为接口，调用时确直接调用这个代码块，这样的效率更高，可读性更强

  2. 方法引用

    1. 概述

      我们有时仅仅用lambda调用了一个已存在的方法。

      Arrays.sort(StringsArrays,(s1,s2)->s1.compareToIgnoreCase(s2));

      从Java8中，我们可以通过方法引用来简写lambda表达式中已存在的方法

      Arrays.sort(StringArrays,String::compareToIgnoreCase);

    2. 什么是方法引用

      方法引用是用来直接访问类或者实例的已经存在的方法或者构造方法。方法引用提供了一种引用而不执行方法的方式，它需要由兼容的函数式接口构成的目标类型上下文。计算时，方法引用会创建函数式接口的一个实例。
